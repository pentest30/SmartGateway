Service Self-Registration

Kubernetes-style service discovery without Consul or etcd. Services register themselves at startup, send heartbeats, and are automatically evicted when they stop.

How It Works

Service Startup SmartGateway YARP Proxy | | | |-- POST /register --------->| | | |-- Add destination to DB | | |-- SignalReload() ----------->| | | |-- Route traffic | | | |-- PUT /heartbeat --------->| (every TTL/2 seconds) | |-- PUT /heartbeat --------->| | | | | | (service crashes) | | | X | | | |-- TtlExpiryService runs | | |-- LastHeartbeat > TTL | | |-- Mark IsHealthy=false | | |-- SignalReload() ----------->| | | |-- Stop routing

API Endpoints

MethodPathDescription
POST/admin/api/services/registerRegister a destination (upsert)
PUT/admin/api/services/{id}/heartbeatRefresh TTL timer
DELETE/admin/api/services/{id}Graceful deregistration
GET/admin/api/servicesList all destinations

Registration

When a service starts, it calls the register endpoint with its cluster, address, and TTL:

Try it: Register a service

curl -X POST http://localhost:5002/admin/api/services/register \
  -H "Content-Type: application/json" \
  -d '{
    "clusterId": "inventory-service",
    "destinationId": "inventory-pod-abc123",
    "address": "http://10.0.2.55:8080",
    "ttlSeconds": 30
  }'

Registration Request Fields

FieldTypeDescription
clusterIdstringWhich cluster to join (must already exist)
destinationIdstringUnique ID for this instance
addressstringThe upstream URL (e.g. http://10.0.2.55:8080)
ttlSecondsintHow long before expiry if no heartbeat. Use 0 for permanent.

If a destination with the same clusterId + destinationId already exists, it updates the address and refreshes the heartbeat (upsert behavior).

Heartbeat

Services must send periodic heartbeats to prevent TTL expiry. The recommended interval is TTL / 2 (e.g., every 15s for a 30s TTL).

Try it: Send heartbeat

curl -X PUT http://localhost:5002/admin/api/services/inventory-pod-abc123/heartbeat

Each heartbeat updates LastHeartbeat to DateTime.UtcNow and sets IsHealthy = true (re-admits if previously evicted).

TTL Expiry

TtlExpiryService (Background Service)

Runs every 5 seconds inside SmartGateway.Host. It queries all destinations where:

TtlSeconds > 0 AND IsHealthy = true AND (now - LastHeartbeat) > TtlSeconds

Matching destinations are marked IsHealthy = false and excluded from YARP routing on the next reload.

Note Destinations with TtlSeconds = 0 are never expired — they are permanent until manually removed.

Graceful Deregistration

When a service shuts down cleanly (e.g., pod termination), it should deregister itself:

Try it: Deregister on shutdown

curl -X DELETE http://localhost:5002/admin/api/services/inventory-pod-abc123

This immediately removes the destination from the database and triggers a YARP reload.

Full Lifecycle Example

1
Service starts

Calls POST /register with TTL=30s

2
Heartbeat loop

Every 15s, calls PUT /heartbeat

3
Service crashes

Heartbeats stop

4
TTL expires (30s later)

TtlExpiryService marks IsHealthy = false

5
Traffic rerouted

Remaining healthy destinations handle all traffic

6
Service recovers

Calls POST /register again — re-admitted immediately

Integration with Your Service

Add this to your .NET service's Program.cs:

// On startup
var http = new HttpClient();
await http.PostAsJsonAsync("http://gateway:5002/admin/api/services/register", new {
    clusterId = "my-service",
    destinationId = Environment.MachineName,
    address = $"http://{GetLocalIp()}:8080",
    ttlSeconds = 30
});

// Background heartbeat
var timer = new PeriodicTimer(TimeSpan.FromSeconds(15));
while (await timer.WaitForNextTickAsync())
    await http.PutAsync($"http://gateway:5002/admin/api/services/{Environment.MachineName}/heartbeat", null);

// On shutdown (IHostApplicationLifetime.ApplicationStopping)
await http.DeleteAsync($"http://gateway:5002/admin/api/services/{Environment.MachineName}");
